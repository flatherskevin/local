# Chapter 5: Crawl, Week 1

Week 1 is about surviving comfortably.

## Your Goal This Week

Your goal is not speed.

Your goal is to stop feeling lost.

If by the end of the week you can open the right repo, find files, search text,
make edits, and run commands beside the editor without friction, the week was a
success.

## Daily Rule

Start work with:

```bash
dev
```

Then keep the layout simple:

- left pane: Neovim
- right pane: shell

Do not improvise a more complex layout until this one feels natural.

## Only Learn These Neovim Basics First

- `i`, `a`, `o`
- `Esc`
- `:w`, `:q`, `:wq`
- `/`, `n`, `N`
- `u`, `Ctrl-r`
- `yy`, `dd`, `p`

That is enough to do real work. Do not overload yourself with advanced motions
yet.

## Only Learn These Workflow Keys First

- `F5`: grep
- `F6`: files
- `F8`: explorer
- `F2`: rename
- `F3`: format
- `F10`: LazyGit

## Success Condition For Week 1

You can:

- open the right repo fast
- find files without hunting
- search text across the repo
- edit and save confidently
- keep tests and commands beside the editor

## End-Of-Day Practice Loop

At the end of each day, do this on purpose once:

1. open a repo with `dev`
2. find a file with `F6`
3. grep for a symbol with `F5`
4. rename something safe with `F2`
5. format with `F3`
6. review git changes with `F10`

That repetition matters more than reading about the setup.
