# Foundations

These four chapters establish the mental model and get the environment running
before you start daily practice.

Read them in order once. They are short on purpose.

## What These Chapters Cover

1. [Getting Oriented](01-getting-oriented.md) -- where to start, what the
   default layout looks like, which keys to learn first.
2. [Mental Model](02-mental-model.md) -- how kitty, tmux, and Neovim divide
   the work, and why that division matters.
3. [Install and Validate](03-install-and-validate.md) -- clone, bootstrap, and
   confirm everything works before you touch real code.
4. [VS Code Bridge](04-vscode-bridge.md) -- translate your daily VS Code
   actions into this workflow so you stop reaching for habits that no longer
   apply.

## When To Move On

Move to Week 1 when you can:

- run `dev` and land in the two-pane layout without thinking
- use F5, F6, and F8 to find things
- explain in one sentence what each layer (kitty, tmux, Neovim) is responsible
  for

You do not need to feel fast. You need to feel oriented.
