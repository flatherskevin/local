# Week 3: Flow

Week 3 is about feeling fast rather than merely functional.

You survived. You navigate. Now you stop thinking about your tools and start thinking about your code. By Friday the terminal setup should feel like your normal working environment. Key presses happen without conscious thought. You open files, edit, format, switch repos, and manage splits without pausing to remember what to press.

## Success Criteria

By the end of this week you should be:

- **Editing with vim motions** -- using ciw, di(, di", and word motions instead of selecting text character by character.
- **Trusting format-on-save** -- writing messy code and letting the formatter clean it up, using F3 and F4 when you need manual control.
- **Working across repos** -- one kitty tab per workstream, switching between projects without losing context.
- **Managing splits and buffers** -- viewing multiple files side by side and switching buffers without closing anything.

## How to Use This Week

One day at a time. The exercises are shorter this week because the goal is speed, not discovery. If something still feels slow, repeat it until it doesn't. The moment you stop thinking about how to do something, you've learned it.
