# Chapter 10: Neovim And LazyVim Workflow

This setup uses LazyVim so you are not maintaining a custom editor from scratch.

## What To Expect

You do not need to become a Vim expert before this becomes useful.

The first win is not exotic motion editing. The first win is faster navigation,
stronger code intelligence, and a cleaner editing loop.

## Most Important Keys

- `F1`: key help
- `F2`: rename
- `F3`: format
- `F4`: code action
- `F5`: grep
- `F6`: files
- `F7`: buffers
- `F8`: explorer
- `F9`: terminal
- `F10`: LazyGit

## Why LazyVim

- large community
- active maintenance
- good defaults
- modular extension model

## What To Learn First

- file finding
- project grep
- LSP navigation
- formatting
- code actions

Do not start with plugin tweaking.

## What To Learn Second

Once the basics feel normal, add:

- hover docs with `K`
- definitions with `gd`
- references with `gr`
- diagnostics with `]d` and `[d`

That is where the editor starts paying you back.
