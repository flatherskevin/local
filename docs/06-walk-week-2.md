# Chapter 6: Walk, Week 2

Week 2 is about turning the setup into a repeatable work system.

## Your Goal This Week

By the end of week 2, you should be using the environment intentionally rather
than simply coping with it.

That means:

- choosing tmux windows on purpose
- relying on LSP navigation
- using search more than visual scanning
- keeping repo context organized

## Start Using Tmux As A Workspace Manager

By now, use:

- one tmux session per repo
- one tmux window per task area

Example:

- window 1: editing and tests
- window 2: server or watcher
- window 3: logs or infrastructure

## Start Leaning On LSP

Practice:

- `gd` for definition
- `gr` for references
- `K` for hover
- `]d` and `[d` for diagnostics

These are the features that make the editor feel strong instead of bare.

## Use The File Explorer Sparingly

Use `F8` for orientation and file operations, not as a permanent sidebar.

Prefer:

- `F6` for file jumps
- `F5` for text search

## End-Of-Week Check

You are ready for week 3 if:

- you can reopen your repo sessions without rebuilding context
- you naturally reach for search and file finding
- you use definitions and references when reading unfamiliar code
- you no longer feel dependent on a permanent sidebar
